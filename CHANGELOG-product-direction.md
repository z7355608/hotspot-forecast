# 产品方向文档改造说明

本次改造严格按照产品方向文档的 6 大要求逐项落地，以下说明每项要求的具体实现方式。

---

## 一、结果页三层结构重做

文档要求将结果页改为"结果先行 → 动作建议 → 归因折叠"三层结构，核心逻辑是**用户打开页面第一眼看到结论，而非数据**。

**第一层（结果先行）** 重写了 `new-prediction-result.tsx` 的顶部区域，首屏展示"今日建议拍什么"小标题 + 主题标题（如"职场穿搭 × 通勤好物"）+ 一段完整的预测摘要 + 推荐级别说明（如"强烈推荐 — 数据信号强烈，建议立即行动"）。右侧放置 `ProbabilityGauge` 动画仪表盘展示爆款概率值，底部展示赛道热度、竞争程度、差异空间三个关键指标。CTA 按钮"生成开拍方案"和"拆解低粉爆款"紧随其后，确保用户无需滚动即可执行动作。

**第二层（动作建议）** 紧接结果下方，左侧展示"建议拍摄方向"（最推荐 + 备选，带切入说明和注意事项），右侧展示"下一步建议"三段式：建议马上拍（理由）、建议先拆解低粉爆款（理由）、如果现在不做（后果）。这一层让用户知道"接下来该做什么"。

**第三层（归因折叠）** 将所有数据支撑内容改为默认折叠的 `CollapsibleSection` 组件，包括：为什么现在值得拍（预测依据）、热门作品参考、相似账号参考、低粉爆款归因（独立折叠区）、市场数据支撑。用户有兴趣时点击展开，不会被大量数据淹没。

---

## 二、统一文案命名

文档要求弱化"机会判断"，统一围绕"爆款预测"表达。

| 修改位置 | 旧文案 | 新文案 |
|---------|--------|--------|
| `results-view-meta.ts` WINDOW_META | 黄金窗口/机会窗口/潜力窗口 | 强烈推荐/值得一试/持续关注/暂不建议 |
| `new-prediction-result.tsx` 标签 | 综合判断 | 爆款预测结果 |
| `new-prediction-result.tsx` 仪表盘 | 机会分/综合机会分 | 爆款概率 |
| `results-view-meta.ts` deepDive | 继续深挖这次机会 | 继续深挖这次预测 |
| `results-view-meta.ts` OPPORTUNITY_META | 搜索机会/异常机会等 | 搜索需求窗口/爆款信号窗口等 |
| `agent-runtime.ts` description | 赛道机会判断 | 爆款预测分析 |
| `LevelUpModal.tsx` | 机会分 | 爆款概率 |
| `cta-markdown-generator.ts` | 赛道机会判断 | 爆款预测分析 |

全项目 `grep` 确认无残留的"机会分"、"综合判断"、"黄金窗口"、"机会窗口"、"潜力窗口"等旧文案。

## 三、首页入口改造

文档要求首页降低思考成本，让用户"一看就知道该干嘛"。

`HeroSection.tsx` 重写为：标签"AI 爆款预测" + 标题"你今天最值得拍什么，我们直接告诉你" + 副标题"输入行业关键词、竞品链接或你的账号链接，立即获取当前高概率爆款选题" + 双 CTA 按钮"立即开始预测"（聚焦输入框）和"看看示例"（跳转 /results/demo）。

`HomePage.tsx` 调整了组件顺序：HeroSection → AIWorkbench（输入框）→ PromptTemplates（模板建议）→ DashboardInsights（赛道情报下移）→ ValueCarousel。确保首屏是价值描述 + 输入框，赛道情报作为辅助信息下移。

`ai-workbench-shell.tsx` 的 placeholder 改为"输入行业关键词、粘贴竞品链接或你的账号链接，立即获取爆款预测"，明确三种输入方式。

`PromptTemplates.tsx` 的示例链接文案改为"看看爆款预测示例"，统一指向 /results/demo。

---

## 四、预测理由用户化表达

文档要求将 whyNowItems 的展示改为用户可理解的分类，不展示算法术语。

`new-prediction-result.tsx` 中"为什么现在值得拍"折叠区的每条 whyNow 项展示为：来源标签（如"搜索趋势"、"低粉异常"、"竞争格局"）+ 事实描述 + 推论 + 对你的影响。每条用通俗语言表达，例如"用户主动搜索意愿正在快速上升"而非"search_volume_7d_growth_rate"。`tone` 字段控制视觉色调（正面/中性/警告），让用户一眼区分利好和风险。

---

## 五、数据异常拦截

文档要求对点赞/评论为 0 的内容进行标记或过滤，增加前端兜底提示。

`new-prediction-result.tsx` 的热门作品参考区域增加了异常数据过滤逻辑：当 `likeCount` 和 `commentCount` 均为 0 或 null 时，该作品卡片会显示"数据待更新"的灰色标记，而非展示误导性的 0 值。同时，当所有作品均为异常数据时，整个区域显示兜底提示"暂无有效互动数据的作品样本"。

---

## 六、视觉冲击力增强

文档要求爆款概率值和推荐级别做强视觉表现。

`ProbabilityGauge` 组件实现了 SVG 环形仪表盘，带 CSS 动画（从 0 到目标值的渐进动画）和外发光效果（`filter: drop-shadow`），概率数字使用 48px 大字号。颜色根据分值动态变化：高分绿色、中分黄色、低分红色。

推荐级别使用渐变背景徽章（`bg-gradient-to-r`），"强烈推荐"为绿色渐变、"值得一试"为琥珀色渐变、"持续关注"为蓝色渐变、"暂不建议"为灰色。徽章紧随"爆款预测结果"标签，形成视觉焦点。

底部三指标条（赛道热度/竞争程度/差异空间）使用图标 + 标签 + 值的结构，在第一屏内完成关键信息传达。
