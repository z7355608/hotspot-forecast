基于我现有产品的设计语言，设计两个连续页面：

1. 低粉爆款页
2. 爆款拆解详情页

这两个页面必须严格延续当前产品的统一视觉体系，不要重新设计品牌，不要换成内容社区或视频平台风格。页面属于一个桌面端中文 Web SaaS 产品，整体风格要求：

- 浅灰白背景
- 左侧固定导航栏
- 顶部横向栏
- 中间主内容区居中
- 白色大圆角卡片
- 轻阴影、细边框、低饱和
- 风格理性、克制、可信、精致
- 页面是“结构化判断 + 样本发现 + 借鉴建议”的产品，不是聊天产品，不是内容社区，不是视频平台页面

这两个页面还必须内建会员与积分逻辑：
- 会员用于控制内容深度、完整样本、高级筛选和个性化能力
- 积分用于解锁高价值动作，例如生成借鉴建议、扩展相似样本、生成标题和提纲
- 权益提示必须轻量、嵌入式，不要强打断，不要大弹窗，不要让页面像支付中心
- 用户必须能清楚区分“会员解锁内容”和“积分解锁能力”

-----------------------------------
第一页：低粉爆款页
-----------------------------------

页面名称：低粉爆款

页面目标：
帮助创作者快速发现中国短视频平台中“低粉账号跑出超预期表现”的内容样本，并判断哪些值得借鉴。

页面结构要求：

1. 页面头部
- 标题：低粉爆款
- 一句说明文案：发现低粉账号中跑出超预期表现的内容样本，快速判断哪些值得借鉴

2. 顶部工具区
- 一个大搜索框，placeholder：搜索关键词、赛道、标题结构、平台
- 一组筛选器：
  - 平台：抖音 / 小红书 / 视频号 / 快手 / B站
  - 粉丝区间：0–1k / 1k–5k / 5k–1w / 1w–5w
  - 内容形式：口播 / 图文 / 剪辑 / 测评 / Vlog / 干货
  - 赛道：职场 / 搞钱 / 情感 / 育儿 / 本地 / 知识 / 美妆
  - 爆发时间：24h / 7天 / 30天
  - 排序：异常倍数最高 / 最近爆发 / 最适合新手借鉴 / 与我方向最接近

3. 会员与筛选逻辑
- 基础筛选对所有用户可见
- 高级筛选和个性化排序对会员开放
- 高级筛选项旁要有轻量“会员”标识，例如：
  - 更适合你的低粉爆款
  - 更低粉丝区间
  - 更长时间窗口
  - 高级平台组合筛选
- 不要用重弹窗拦截，优先用嵌入式灰态、lock icon、轻提示文案

4. 顶部推荐模块
- 模块标题：今日值得参考的低粉爆款
- 展示 3 张稍大的重点样本卡片
- 其中至少有 1 张体现“会员可查看完整拆解”状态

5. 主内容区
- 采用 3 列卡片栅格
- 每张卡片是“低粉爆款样本卡”
- 卡片包含：
  - 封面图
  - 视频时长角标（如适用）
  - 平台图标
  - 异常倍数 badge，例如 4.2倍
  - 粉丝数 badge，例如 3200粉
  - 标题
  - 账号名
  - 内容方向标签
  - 表现数据，例如 当前播放、超近30条均值
  - 爆因标签，例如：新人视角、标题切口强、平台近期偏好
  - 一句话借鉴判断，例如：更适合新手借鉴切角，不建议直接照搬形式
  - 底部动作区：
    - 查看拆解
    - 生成借鉴建议 30积分

6. 会员内容深度控制
- 免费用户可浏览基础样本池和基础卡片信息
- 会员用户可查看更多样本、完整个性化推荐区、完整拆解入口、高级筛选
- 在样本流中插入轻量升级卡片，而不是突然中断
- 升级卡文案示例：
  - 解锁更多低粉爆款样本
  - 查看完整拆解与个性化排序
  - 开通会员

7. 个性化模块
- 在主内容区中加入一行“更适合你的低粉爆款”
- 该模块带会员标识
- 未开会员时可显示预览态：模糊 1–2 张卡片 + 解锁提示

8. 积分动作逻辑
- 样本卡上的“查看拆解”是内容访问动作，优先由会员控制完整度
- 样本卡上的“生成借鉴建议”是高价值能力动作，使用积分解锁
- 卡片动作区要明确区分：
  - 查看拆解（会员内容）
  - 生成借鉴建议 30（积分能力）
- 积分按钮要清晰显示消耗，不要隐藏

-----------------------------------
第二页：爆款拆解详情页
-----------------------------------

页面名称：爆款拆解

页面目标：
让用户理解这条低粉爆款内容为什么会异常跑出、哪些点值得借鉴、哪些不建议照搬，并能进一步生成自己的内容建议。

页面结构要求：

1. 顶部样本摘要区
- 展示封面图 / 样本缩略图
- 标题
- 账号名 + 粉丝数
- 平台
- 发布时间
- 核心数据：播放 / 点赞 / 收藏 / 评论
- 异常倍数
- 一句话结论，例如：这条内容异常跑出的关键在于“职场新人视角 + 强痛点标题”的组合，而不是单纯教程内容
- 右侧展示 3 到 4 个核心判断指标：
  - 异常值
  - 可借鉴度
  - 适合对象
  - 风险度

2. 爆因拆解区
- 模块标题：它为什么能爆
- 用 3 到 4 张结构化卡片拆解：
  - 选题切口
  - 标题结构
  - 开头钩子
  - 表达方式 / 情绪触发
  - 平台时机（可选）
- 每张卡片包含：
  - 核心判断
  - 简短说明
  - 证据点

3. 值得借鉴 / 不建议照搬区
- 左侧模块：值得借鉴
- 右侧模块：不建议照搬
- 每侧用条目化结构展示

4. 适合谁参考
- 模块标题：更适合哪些账号借鉴
- 展示：
  - 更适合新手账号 / 成长期账号
  - 更适合抖音口播 / 小红书图文
  - 更适合哪些赛道

5. 相似样本区
- 展示 3 个相似样本卡片
- 用于证明这不是孤例，并给出更多参考方向

-----------------------------------
详情页里的会员逻辑
-----------------------------------

1. 免费用户可看：
- 顶部样本摘要区
- 一句话结论
- 部分爆因拆解
- 部分“值得借鉴”
- 相似样本预览

2. 会员用户可看完整版：
- 完整爆因拆解
- 完整“值得借鉴 / 不建议照搬”
- 完整“适合谁参考”
- 完整相似样本
- 更深的判断层信息

3. 会员锁内容的呈现方式：
- 模块内轻量遮罩
- “开通会员查看完整拆解”
- “解锁完整相似样本与借鉴判断”
- 不要全页弹窗，不要强打断

-----------------------------------
详情页里的积分逻辑
-----------------------------------

1. 页面底部行动区
- 主按钮：生成我的借鉴建议 20积分
- 次按钮：按我的账号方向重写这个切口 20积分
- 辅助按钮：生成类似标题 30积分 / 生成开头钩子 30积分 / 输出内容提纲 30积分

2. 积分动作的表达方式
- 必须表现为“能力解锁”，不是单纯按次提问收费
- 例如：
  - 解锁借鉴建议
  - 按你的账号方向改写
  - 生成可直接使用的标题和提纲

3. 点击带积分动作后，使用轻量提示条或轻确认条：
- 本次将解锁：借鉴建议
- 预计消耗 20 积分
- 当前余额 120 积分
- 确认生成 / 取消

4. 积分不足时
- 不要全屏拦截
- 在按钮附近或行动区下方显示轻量不足提示
- 文案示例：
  - 当前积分不足
  - 还差 20 积分
  - 去充值 / 开通会员
  - 或先查看免费内容

-----------------------------------
整体 UX 原则
-----------------------------------

1. 低粉爆款页负责：发现异常样本
2. 爆款拆解详情页负责：解释爆因与借鉴价值
3. 会员负责：解锁更深内容
4. 积分负责：解锁更高价值动作
5. 用户必须始终清楚：
   - 我当前看到的是免费内容、会员内容，还是积分动作
   - 我付费后能获得什么
   - 我不是在为页面付费，而是在为“下一层判断和执行能力”付费

视觉要求：
- 必须与现有首页、结果页、追问页统一
- 中文高保真 Web SaaS 产品设计稿
- 白色圆角卡片、浅灰背景、轻阴影、低饱和
- 封面图可以强视觉，但信息区必须规整、结构化
- 不要像抖音网页版
- 不要像内容社区
- 不要像 YouTube 工具站
- 不要像支付中心
- 要更像“样本发现 + 爆因判断 + 借鉴建议”的工作台
- 适合 Figma 落地
