请设计“带积分消耗逻辑的继续追问结果页”，要求如下：

一、追问入口区
1. 在结果页底部新增“继续追问模块”，页内展开，不跳页，不做弹窗，不做全屏遮罩。
2. 模块标题为：基于当前结果继续追问
3. 副文案为：换个条件重新判断，或直接进入执行建议
4. 有一个紧凑版输入框，风格延续首页输入框，但更短更轻。
5. 输入框 placeholder 示例：
   - 如果我是新号呢？
   - 这个方向具体怎么切？
   - 给我 3 个适合抖音的标题
6. 输入框下方展示快捷追问标签，每个标签都带积分标识。
7. 快捷追问标签分三类：
   - 解释型：更适合新手吗（免费）、为什么竞争度偏高（5）
   - 修正型：再给我两个类似账号（20）、换成小红书版（20）
   - 执行型：生成标题建议（30）、生成开头钩子（30）

二、积分消耗提示逻辑
1. 当用户点击带积分的追问标签后，在输入框下方显示一条轻量积分提示条，不使用重弹窗。
2. 积分提示条内容包括：
   - 本次将解锁的能力，例如：新手视角判断 / 样本扩展 / 标题建议
   - 预计消耗积分，例如：预计消耗 10 积分
   - 当前余额，例如：余额 120 积分
3. 轻量追问可以直接执行，不需要强确认。
4. 中高积分追问显示轻确认条：
   - 本次将消耗 30 积分，生成 3 个标题方向
   - 按钮：确认生成 / 取消
5. 积分逻辑表达要像“能力解锁”，不要像简单按次收费。

三、主结论更新机制
1. 用户提交追问后，不要整页刷新。
2. 在原有综合判断模块附近增加一条轻量“主结论更新条”。
3. 更新条文案示例：
   - 已按“新手账号”视角补充判断
   - 当前更推荐从“职场新人 Excel 入门”切入
   - 已切换为“小红书表达”版本
4. 这条更新条用于让用户感觉系统在校准当前结论，而不是无限补答案。

四、追问结果展示
1. 追问结果不要做成聊天气泡，要保持结构化信息卡片风格。
2. 追问结果卡分为两种规格：
   - 轻追问结果卡：用于解释型和轻修正型问题，信息更短、更密
   - 重追问结果卡：用于执行型问题，内容更完整
3. 轻追问结果卡结构：
   - 标题：补充判断
   - 副标签：针对“更适合新手吗”
   - 一句核心回答
   - 2 条简短理由
   - 底部两个动作：继续细化方向 / 进入执行建议
4. 重追问结果卡结构：
   - 标题：执行建议
   - 副标签：针对“生成标题建议”
   - 一句引导说明
   - 3 条结构化结果，例如标题建议
   - 底部动作：生成开头钩子 / 输出内容提纲
5. 每张结果卡底部都显示一次轻量反馈：
   - 已消耗 10 积分
   - 已新增：新手视角判断
   - 或：已消耗 30 积分，已生成标题建议

五、积分不足状态
1. 如果用户积分不足，不要强拦截整页，不要重弹窗。
2. 在输入区域下方显示轻量不足提示卡：
   - 当前积分不足
   - 还差 20 积分
   - 可先使用免费追问
   - 去充值 / 开通会员
3. 提供替代路径：
   - 使用免费轻追问
   - 或开通会员解锁更深追问能力

六、整体交互原则
1. 追问后的页面目标不是堆更多答案，而是让结果更收敛。
2. 页面要体现两条明确路径：
   - 继续判断

3. 追问结果要比首轮结果更短、更密、更聚焦。
