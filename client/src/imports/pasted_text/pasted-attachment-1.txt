基于我已经设计完成的“爆款拆解详情页”现有设计稿，做一版优化版设计。不要重新发明新页面，不要改变整体布局框架和当前产品的统一视觉体系。请在现有结构基础上优化信息层级、会员与积分边界、模块优先级和转化路径。

已知当前页面特征：
- 桌面端中文 Web SaaS 产品
- 左侧固定导航栏
- 顶部横向栏
- 浅灰白背景
- 白色大圆角卡片
- 轻阴影、细边框、低饱和
- 风格理性、克制、可信、精致
- 页面已有：
  - 首屏样本摘要区
  - 一句话判断
  - 核心数据与可借鉴度评分
  - 它为什么能爆
  - 值得借鉴 / 不建议照搬
  - 更适合哪些账号借鉴
  - 相似样本参考
  - 生成你自己的内容策略
- 当前目标不是重做，而是优化为“价值边界清晰的可付费拆解详情页”

优化要求如下：

一、首屏摘要区
1. 保留当前大卡布局：样本媒体、标题、账号、平台、时间、数据、评分、一句话判断。
2. 优化一句话判断，使其仍然有结论性，但不要把完整会员价值提前讲透。
3. 在首屏区域加入一条轻量权益说明，清楚区分：
   - 会员解锁完整拆解与完整适配判断
   - 积分用于生成专属借鉴建议
4. 这条说明要轻量、嵌入式，不要像弹窗，不要破坏首屏简洁感。

二、“它为什么能爆”模块
1. 保留现有模块结构和双列卡片逻辑。
2. 每张拆解卡统一结构：
   - 核心判断
   - 简短解释
   - 证据点
3. 证据点要更显性，例如：
   - 近30天同类切口平均播放更高
   - 含明确数字的标题 CTR 更高
4. 整体提升这块的“可信度”和“证据感”。

三、“值得借鉴 / 不建议照搬”模块
1. 保留左右对照结构。
2. 这组模块要成为会员价值最强的一部分。
3. 免费用户只预览少量内容，完整条目对会员开放。
4. 锁定文案不要只是“查看完整内容”，而要强调：
   - 查看完整借鉴边界判断
   - 明确哪些值得学、哪些不要照搬
5. 视觉上要让用户感觉这是高价值判断，不是普通附加内容。

四、“更适合哪些账号借鉴”模块
1. 保留账号阶段、平台/形式、适合赛道等现有结构。
2. 将模块语义从静态信息展示优化成“适配判断”。
3. 增强这块的价值表达：
   - 查看更完整的匹配分析
   - 减少照搬误判
   - 结合账号阶段、平台和内容风格判断是否值得借
4. 保持会员锁定逻辑，但强调收益而不是权限。

五、“相似样本参考”模块
1. 保留样本卡片形式。
2. 强化这块的证据价值，而不是做成普通推荐区。
3. 模块标题或辅助说明要体现：
   - 证明这不是孤例
   - 同类切口在近期低粉样本中重复成立
4. 每张相似卡片增加“相似点”小标签，例如：
   - 同样使用新人视角
   - 同样依赖强问题标题
   - 同样适合职场赛道
5. 免费展示少量样本，更多相似样本对会员开放。

六、“生成你自己的内容策略”模块
1. 保留当前积分动作区，但优化为更明确的转化路径。
2. 明确一个主动作：
   - 生成我的借鉴建议 20积分
3. 其他动作作为第二层扩展：
   - 按我的方向重写切口 20积分
   - 生成类似标题 30积分
   - 生成开头钩子 30积分
   - 输出内容提纲 30积分
4. 在动作区增加一条轻量推荐说明：
   - 建议先生成借鉴建议，再继续扩展标题和提纲
5. 每个动作都增加一条轻量解释，让用户知道积分解锁的具体结果：
   - 生成我的借鉴建议：基于样本生成适合你账号方向的策略
   - 按我的方向重写切口：保留爆因结构，替换成你的赛道表达
   - 生成类似标题：输出可直接使用的标题方向
6. 积分不足时，除了充值/开会员，也加入更轻的替代路径提示，例如：
   - 先继续查看免费内容
   - 先收藏样本
   - 推荐优先完成的低积分动作

七、整体原则
1. 页面目标不是给出更多内容，而是让用户更快理解：
   - 这条内容为什么值得借
   - 哪些不能照搬
   - 适不适合自己
   - 下一步最值得花积分生成什么
2. 页面中必须清楚区分三层价值：
   - 免费：判断这条值不值得继续看
   - 会员：查看完整拆解与完整适配判断
   - 积分：把样本转成自己的内容策略
3. 不要新增聊天式布局，不要做成长文文章页，不要做成支付中心。
4. 必须保留当前产品统一的结构化判断产品感。

视觉要求：
- 与现有页面高度统一
- 中文高保真 Web 产品设计稿
- 白底圆角卡片、浅灰背景、轻阴影、低饱和
- 信息层级更清楚，主次更明确
- 保持简洁克制，不要过度装饰
- 适合 Figma 落地
